# LeetCode Interview Prep

这个仓库用来记录我的大厂面试刷题准备过程。

目标不是堆题量，而是把每一道题沉淀成三件事：

1. 我识别到了什么题型。
2. 我用了什么数据结构维护状态。
3. 下次遇到相似题时，我如何更快想出解法。

## 当前目标

- 时间：2026-07-07 至 2026-08-31，8 周。
- 主线：LeetCode 面试热题 150。
- 方式：每天约 2 小时，按考点块推进。
- 留痕：每天提交代码、复盘、错误记录。

## 目录结构

```text
plans/       刷题路线图和每日安排
solutions/   题解代码，按题型分类
notes/       模式总结、错误本、Python 工具
daily-log/   每日刷题日志
```

## 每日流程

1. 10 分钟：回忆昨天错点，写出今天题型的核心结构。
2. 20 分钟：学习或复习当天所需的数据结构。
3. 70 分钟：独立写主线题。卡住时先画状态，不急着看答案。
4. 15 分钟：重写关键片段或补一题轻量题。
5. 5 分钟：写当天心得并提交 Git。

## 代码纪律

- 先说变量职责，再写代码。
- 先跑小样例，再提交。
- 不把别人的最优解当作自己的能力。
- 每道错题必须写一句「我卡住的真正原因」。
- 学到的新 Python 操作要记录到 `notes/python-tools.md`。
