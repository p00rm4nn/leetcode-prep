# 模式总结

## 链表

链表题不要先写代码，先画指针。

常见变量职责：

| 变量 | 含义 |
| --- | --- |
| `dummy` | 虚拟头节点，简化头节点变化 |
| `cur` | 当前构造位置或扫描位置 |
| `prev` | 当前节点前一个节点 |
| `slow` / `fast` | 快慢指针，用于环、倒数第 N 个、找中点 |
| `carry` | 加法链表中的进位 |

核心问题：

1. 当前节点处理完后，谁往前走？
2. 有没有断链风险？
3. 返回的是 `head` 还是 `dummy.next`？

## 栈

栈题的第一问永远是：栈里存什么？

常见类型：

| 类型 | 栈里存什么 |
| --- | --- |
| 括号匹配 | 未匹配的左括号，或期待的右括号 |
| 路径简化 | 有效路径段 |
| 最小栈 | 当前值 + 当前最小值 |
| 逆波兰表达式 | 等待参与运算的数字 |
| 单调栈 | 尚未找到答案的元素或下标 |
